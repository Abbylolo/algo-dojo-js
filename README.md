## 一、动态规划（Dynamic Programming）

### 1. 基本概念

动态规划是一种通过将问题**分解为更小的子问题**，逐步求解并**存储中间结果**，从而**避免重复计算**的优化方法。

### 2.核心特点

1. **最优子结构**：问题的最优解包含子问题的最优解
2. **重叠子问题**：子问题在求解过程中会重复出现
3. **状态转移**：从已知状态推导出未知状态

### 3. 解题步骤

1. **确定状态**

   - 研究最优解的最后一步
   - 化为子问题
2. **确定dp数组含义**

   - dp[i] 表示什么
3. **确定状态转移方程**

   - 找到状态之间的关系
   - 写出推导公式
4. **确定初始状态**

   - 确定边界条件
   - 设置初始值
5. **确定遍历顺序**

   - 从前往后还是从后往前
   - 是否需要考虑依赖关系

### 4. 经典例题

* 53  最大子数组和
* 121 买卖股票的最佳时机

## 二、贪心算法（Greedy Algorithm）

### 1. 基本概念

贪心算法是一种在每一步选择中都采取**当前状态下最好或最优的选择**，从而希望导致结果是最好或最优的算法。

### 2. 核心特点

- 贪心选择性质：每一步都做出当前看来最好的选择
- 无后效性：当前决策只与当前状态有关，不依赖之前的选择
- 子问题最优性：局部最优解能导致全局最优解

### 3. 解题步骤

1. 建立数学模型来描述问题
2. 把求解的问题分成若干个子问题
3. 对每一子问题求解，得到子问题的局部最优解
4. 把子问题的解局部最优解合成原来问题的一个解

### 4. 适用场景

- 问题能够分解成子问题
- 子问题的最优解能递推到最终问题的最优解
- 对每个子问题的解决方案都能得到局部最优解
- 局部最优解能导致全局最优解

### 5. 经典例题

* 53  最大子数组和

### 6.贪心算法与动态规划算法区别

1. **决策方式**
   - 贪心：局部最优选择
   - 动态规划：通过子问题的最优解推导出全局最优解
2. **解的保证**
   * 贪心：不保证得到全局最优解
   * 动态规划：保证得到全局最优解
3. **状态记录**
   * 贪心：通常只需要记录当前状态
   * 动态规划：需要记录所有或必要的子问题解
